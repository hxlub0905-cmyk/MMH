【問題描述】
載入 KLARF 模板並連動 Measurement 資料後，按下執行輸出顯示「已輸出 0 筆」，
且輸出的 KLARF 檔案沒有任何座標修改。

================================================================================
【根因分析過程】

Step 1：釐清輸出為 0 的所有可能原因（依可能性排序）

  原因 1（確認為主因）：KLARF 格式不相容
    - 現有 parser (klarf_parser.py) 只支援「平坦式 KLARF 1.8」
    - 實際使用的是「巢狀階層式 KLARF」（KLA Tencor 工具格式）
    - 關鍵差異：
        * 平坦式：DefectList N col1 col2 … → EndOfList
        * 階層式：List DefectList { Columns 42 { int32 COL, … } Data N { rows } }
    - 因為 parser 找不到 DefectList，defects = []，所以輸出必然 0 筆

  原因 2：_get_image_size 回傳 None
    - 即使 stem 匹配，若 image_path 在目前環境不存在（換機器、路徑變更），
      cv2.imread 讀不到圖，defect 被加到 unmatched，不進 matched

  原因 3：Image stem 命名不一致
    - KLARF 內的 _image_filename 與 batch 跑的 image_path stem 不同

  原因 4：measurements 為空
    - Batch 全部 FAIL → lookup 空 → 全部 unmatched

  原因 5：MeasurementRecord.from_dict 靜默失敗
    - try/except 靜默跳過，不進 lookup

Step 2：使用者貼出真實 KLARF 格式確認
  使用者貼出的格式：
    Record FileRecord  "1.8"
    {
      Record LotRecord "N9U179.1S"
      {
        Record WaferRecord "14"
        {
          List DefectList
          {
            Columns 42 { int32 DEFECTID, int32 XREL, int32 YREL, ...
                         ImageList IMAGEINFO, ... }
            Data 3
            {
              257 7672524 13692865 … Images 1 { "1.000_00001.jpg" "JPG" 1 "23" } … ;
            }
          }
        }
      }
    }

  確認的格式差異（舊 parser vs 實際格式）：
    1. 舊：startswith("DefectList") → 實際：List DefectList（找不到）
    2. 舊：欄位名直接列出 → 實際：Columns N { type COL, type COL, … }（帶型別前綴）
    3. 舊：Data N + EndOfList → 實際：Data N { } 用大括號
    4. 舊：Image 1 { "file" }  → 實際：Images 1 { "file" "JPG" 1 "23" }（複數且有更多欄位）

【座標換算備忘】

  影像座標系：左上角原點，Y 軸向下
  KLARF 座標系：die 左下角原點，Y 軸向上
  → 兩者 Y 軸方向相反

  換算公式：
    dx_px = center_x - W/2
    dy_px = center_y - H/2
    dx_nm = dx_px * nm_per_pixel
    dy_nm = dy_px * nm_per_pixel
    XREL_new = XREL_orig + dx_nm   （X 方向同向）
    YREL_new = YREL_orig - dy_nm   （Y 方向反向，減號必要）
