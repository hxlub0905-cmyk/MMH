STEP 1 — 建立 KLARF Parser（src/core/klarf_parser.py）
─────────────────────────────────────────────────────
  新增 KlarfParser 類別，以 token-scan（狀態機）方式解析 KLARF 1.8 格式。

  設計決策：
  ・_tokenise()：以空白分割，保留引號字串為單一 token
  ・_parse_by_lines()：逐行判斷 section 邊界，分派至對應解析邏輯
  ・_parse_defect_rows()：累積 token 直到 ";" 為止（支援跨行資料列）
  ・_map_row_tokens()：將 flat token list 對應至 column header，
    特別處理 "Image N { … }" block，提取 _image_filename 以供後續匹配

  輸出結構（dict）：
  ・raw_sections        — 非 DefectList 的各 section 原始文字（保留換行 fidelity）
  ・defect_columns      — DefectList header 的 column 名稱列表
  ・defects             — 每筆 defect 為一個 dict（以 column name 為 key）
  ・image_count         — IMAGECOUNT 欄位值
  ・data_count          — "Data N" 列的值
  ・defect_list_header_raw — DefectList 表頭原始文字

  核心方法：
  ・write(parsed, defects, output_path)  — 公開 API，執行原子寫入
  ・_build()                             — 重組各 section 文字

STEP 3 — 建立 KLARF Exporter（src/core/klarf_exporter.py）
────────────────────────────────────────────────────────────
  新增 KlarfTopNExporter 類別，整合 Parser + Writer 並加入業務邏輯。
  （原始規格名稱為 KlarfMinYcdExporter，實作時改為更通用的 KlarfTopNExporter
    以支援 ascending / descending 兩種排序方向）

  八步驟 Pipeline：
    Step 1 — 解析輸入 KLARF（KlarfParser）
    Step 2 — 從 batch_run 建立 stem → 量測值 查詢表（_build_lookup）
    Step 3 — 對每筆 defect 以 _image_filename 的 stem 匹配量測結果
    Step 4 — 讀取影像尺寸（cv2.imread，dict 快取）
    Step 5 — 座標換算（見下方⚠️說明）
    Step 6 — 依 calibrated_nm 排序，取前 N 筆
    Step 7 — 更新 XREL / YREL，建立 preview_rows
    Step 8 — KlarfWriter 寫出新 KLARF（dry_run=True 時跳過）

  ⚠️ 座標換算關鍵規則（Y 軸翻轉，符號不可隨意更改）：
  ─────────────────────────────────────────────────────
  影像座標系：左上角原點，Y 向下為正
  KLARF 座標系：左下角（die corner）原點，Y 向上為正
  兩者 Y 軸方向相反！

  計算步驟：
    dx_px = center_x - W / 2        ← pixel 偏移量（以影像中心為基準）
    dy_px = center_y - H / 2

    dx_nm = dx_px * nm_per_pixel     ← 換算 nm
    dy_nm = dy_px * nm_per_pixel

    XREL_new = XREL_orig + dx_nm     ← X 軸同向，加號 ✓
    YREL_new = YREL_orig - dy_nm     ← Y 軸翻轉，減號 ✓（勿改為加號！）

  nm_per_pixel 推導：
    由 MeasurementRecord 的 calibrated_nm / raw_px 計算（raw_px > 0 時）

  返回 dict：
    exported_count    — 實際輸出筆數
    unmatched_count   — 找不到對應影像的 defect 數
    unmatched_images  — 找不到對應影像的 stem 列表
    min_ycd_nm        — 輸出集合中最小 CD 值
    max_ycd_nm        — 輸出集合中最大 CD 值（第 N 筆）
    output_path       — 輸出路徑
    preview_rows      — 預覽表格資料（list of dict）

STEP 8 — nm/px 統一整合（6 個檔案修改）
─────────────────────────────────────────
  背景：原本 nm/px（nm per pixel）設定分散在 BrowseWorkspace 的
  CalibrationManager，Batch 執行時所有影像共用同一個全域值，
  無法針對不同 Dataset 使用不同校正值。

  解決方案：將 nm_per_pixel 整合進 MeasurementRecipe，讓每個
  Dataset 攜帶自己的校正值，從 Recipe 到 ImageRecord 全程傳遞。

  資料流：
    Recipe 儲存 nm_per_pixel
      → Measure / Batch workspace 選擇 Recipe 時自動帶入 spinbox
        → 使用者可臨時覆蓋 spinbox 值
          → ImageRecord.pixel_size_nm 全程攜帶正確值
            → KlarfTopNExporter 直接從量測記錄推導使用

  8-1. src/core/recipe_base.py（MeasurementRecipe）
  ・新增頂層 dataclass 欄位：nm_per_pixel: float = 1.0
  ・to_dict()   → 加入 "nm_per_pixel": self.nm_per_pixel
  ・from_dict() → nm_per_pixel=float(d.get("nm_per_pixel", 1.0))
    （舊 JSON 不含此欄位時 fallback 為 1.0，向下相容）

  8-2. src/core/recipe_registry.py
  ・create_default_cmg() 明確傳入 nm_per_pixel=1.0

  8-3. src/core/recipes/cmg_recipe.py
  ・_card_to_descriptor() 從 card dict 讀取
    nm_per_pixel=float(card.get("nm_per_pixel", 1.0))

  8-4. src/gui/workspaces/recipe_workspace.py
  ・Identity 卡片（QFormLayout）新增一列：
    label:  "nm / pixel:"
    widget: QDoubleSpinBox（範圍 0.0001–10000.0，4 小數，suffix "nm/px"）
  ・_load_descriptor_to_form()：帶入 desc.nm_per_pixel
  ・_save_recipe()：傳入 nm_per_pixel=self._nm_per_pixel_spin.value()
  ・_new_from_cmg_template()：nm_per_pixel=1.0

  8-5. src/gui/workspaces/measure_workspace.py

  ・_on_recipe_combo_changed()：讀取 desc.nm_per_pixel，

  ・_run_with_recipe_impl()：改用 self._nm_px_spin.value()（原為 CalibManager）
