TASK 1 ─ 獨立影像品質檢測工具 (tools/image_quality_checker.py)
────────────────────────────────────────────────────────────────────────────────

【需求】
  跑大量 batch 時，模糊影像（Resolution/Sharpness 不足）會讓 CD 量測結果偏短。
  需要一個可獨立執行的工具，先對多組影像進行品質篩選，確認閾值後再整合進主流程。

【實作】
  新增 tools/image_quality_checker.py（PyQt6 獨立視窗程式）

  品質指標：
    1. Laplacian Variance  — 主要指標，數值越高代表影像越清晰
    2. Tenengrad           — Sobel 梯度平均平方，輔助指標
    3. FFT HF Ratio        — 高頻能量佔比，輔助指標

  UI 功能：
    - 可加入多個來源資料夾，支援遞迴掃描子目錄
    - 三個閾值可獨立設定（0 = 停用），按「Re-apply」即時重算 PASS/FAIL
    - 結果表格：PASS 綠字、FAIL 紅底紅字，支援欄位排序
    - 狀態列顯示 Total / PASS / FAIL 數量與比例
    - 可匯出 CSV 保存結果

  執行方式：
    python tools/image_quality_checker.py

【後續追加修改 — medianBlur 前處理】
  上司指示：計算指標前須先做
    img = cv2.medianBlur(src, ksize=5)

  原因：SEM 影像常含 salt-and-pepper noise（孤立亮暗點），若不處理，
  Laplacian / Tenengrad 會將這些噪點誤判為高頻邊緣，導致模糊影像的分數虛高。
  medianBlur 以 5×5 中位數取代原值，對極端值免疫，不影響真實邊緣。

  修改位置：tools/image_quality_checker.py → compute_quality()
    (在所有指標計算之前加入此行)

TASK 4 ─ 品質檢測整合進主量測流程
────────────────────────────────────────────────────────────────────────────────

【需求】
  standalone 工具確認 Laplacian 指標有效後，整合進主流程。
  條件：Laplacian var >= 140 才算通過，僅保留 Laplacian 指標（移除其他輔助指標）。

【實作】

  新增 src/core/image_quality.py：
    DEFAULT_LAP_THRESHOLD = 140.0

    def check_lap_quality(img: np.ndarray) -> float:
        denoised = cv2.medianBlur(img, 5)
        lap = cv2.Laplacian(denoised.astype(np.float64), cv2.CV_64F)
        return float(lap.var())

  修改 src/core/measurement_engine.py：
    _make_worker_args()：
      - 新增 quality_lap_threshold 參數（None → 使用 DEFAULT_LAP_THRESHOLD）
      - 將閾值序列化進 args dict，傳入子程序

    _worker_run_image()：
      - 影像載入後、CMGRecipe 執行前，插入 Quality Gate：
          score = check_lap_quality(raw_img)
          result["quality_score"] = round(score, 2)
          if score < threshold:
              result["status"] = "FAIL"
              result["error"] = "Low image quality: Laplacian var=X.X < threshold Y.Y"
              return result   ← 直接返回，不跑 recipe
      - PASS 的影像照常執行完整 pipeline

    run_batch()、run_multi_batch()：
      - 新增 quality_lap_threshold: float | None = None 參數並往下傳遞

  修改 src/gui/workspaces/batch_workspace.py：
    _BatchWorker、_MultiBatchWorker：
      - 新增 quality_lap_threshold: float | None = None 參數
      - 傳入 engine.run_batch / engine.run_multi_batch

【TPT 影響分析】
  - medianBlur(5) + Laplacian.var() 耗時 < 5ms（完整 pipeline 約 50–500ms）
  - PASS 影像：額外開銷 < 1–3%，可忽略
  - FAIL 影像：跳過整個 recipe，**節省時間**
  - 在子程序（ProcessPoolExecutor worker）內執行，不增加 IPC 開銷
  - 整體 TPT 影響：微乎其微，若模糊影像比例高反而更快
