二、Phase F — Auto X-Projection Strip Mask（自動欄位偵測）
────────────────────────────────────────────────────────────────────────────────

【F1 — Two-Pass X-Proj 初版（後被替換）】
  commit: f137fab

  問題背景：
  - 使用者需要在 Batch 模式下自動偵測 MG（Metal Gate）欄位的 X 座標
  - 手動模式（Start X + Pitch）在單張圖上完美運作，但 Batch 時每張圖
    Start X 不同，無法通用
  - 實作了 detect_mg_column_centers_twopass() 兩階段峰值偵測

  問題發現：
  - Two-pass 仍然對每個峰值獨立判斷，PEPI（Epitaxial S/D）的橫向結構
    會使中間欄位的峰值偏移，Auto-detect 框出的 ROI 不準確

【F2 — Pitch-Anchored 相位偵測（正確解法）】
  commit: 0d6604e

  使用者的核心洞見：
  - MG 有固定寬度 + 固定 pitch → X-proj 是已知週期的週期信號
  - 唯一未知量 = 起始 X 位置（相位偏移 φ）
  - Auto-detect 只需找 φ，其餘由 pitch 決定

  解法 detect_mg_column_centers_pitch_phase()：
  1. 計算 X-proj（含 edge-padded smoothing，避免邊界零填充偽峰）
  2. 將 x_proj 補零到 pitch 整數倍，reshape 成 (N, pitch_px) 矩陣
  3. 對 axis=0 取平均 → phase_means[φ] 代表以 φ 為起點的平均信號強度
  4. argmax(phase_means) → 全局最優相位 φ（PEPI 貢獻被平均消除）
  5. 生成中心 = φ, φ+pitch, φ+2·pitch, ...（天然等間距）
  6. 高度閾值 + edge_margin 過濾

  優點：
  - 全局最優，PEPI 對個別峰值的偏向被平均掉
  - 輸出天然是等間距正規網格（與手動模式一致）
  - 比 two-pass 更簡單（單 pass）

  修改範圍：
  - src/core/mg_detector.py — 新增 pitch_phase，移除 twopass
  - src/core/recipes/cmg_recipe.py — call site 更新
  - src/gui/workspaces/measure_workspace.py — call site 更新
  - src/gui/batch_dialog.py — call site 更新

【G2-3 — Batch Run 早期進度提示】

  背景：
  - Run Batch 後，ProcessPoolExecutor 建立 worker process 期間（數秒）
    UI 完全無反應，使用者誤認為凍結

  解法：
  - batch_dialog.py _BatchWorker.run()：在建立 ProcessPool 前
    先 progress.emit(0, total, "Preparing N job(s)…")
  - 所有 job submit 完成後再發 "Submitted N job(s), waiting for results…"
  - 初始 label 由 "Starting…" 改為 "Initializing…"

【G2-4a — Report HTML 匯出 matplotlib 容錯】

  背景：
  - Report → Export HTML → 崩潰，錯誤訊息提示安裝 pandas
  - 實際根因：matplotlib 未安裝（pandas 有安裝），直方圖生成時 import 失敗

  解法：
  - report_generator.py _histogram_b64()：
    將 import matplotlib 包在 try/except ImportError 中
    無 matplotlib 時回傳 "" 空字串
  - HTML 模板：hist_b64 為空時改顯示
    '<p style="color:#888">Histogram unavailable (install matplotlib)</p>'
  - HTML 報告其他部分（統計表格等）不受影響，仍可正常匯出

【Bug 1 — Range Filter 在 Batch/Recipe 路徑失效】

  根本原因：
  - range_enabled / min_line_px / max_line_px 儲存在 detector_config
  - CMGRecipe.compute_metrics() 只讀 edge_locator_config，從未讀 detector_config
  - 因此透過 Recipe 跑 Batch 時，range filter 完全未被套用

  修改（src/core/recipes/cmg_recipe.py compute_metrics()）：
  - 在 X-CD 座標反轉之後、建立 MeasurementRecord 之前插入 range filter 邏輯
  - 過濾：保留 min_px <= cd_px <= max_px 的量測（0 = 停用該端）
  - 過濾後重新計算 MIN/MAX flag
  - context["cmg_cuts"] 在過濾後更新，確保 render_annotations 也只繪製
    過濾後的量測

  影響路徑：
  - 過去有效：Measure Legacy Cards 路徑（measure_workspace.py）
              Batch Legacy Cards 路徑（batch_dialog.py）
  - 現在修復：Recipe 路徑（batch_workspace → engine → CMGRecipe）

【Bug 2 — Review 頁缺少 Overlay 控制勾選框】

  根本原因：
  - Review workspace 只有 Raw/Mask/Annotated 模式切換按鈕
  - 缺少 Lines / Values / Boxes / Legend 勾選框（Measure 頁有）

  修改（src/gui/workspaces/review_workspace.py）：
  - __init__：新增 _cur_raw / _cur_mask / _cur_cuts 快取屬性
  - _build_header()：新增 _overlay_widget（含四個勾選框）
    只在 Annotated 模式顯示（_on_mode_ann/raw/mask 控制）
  - 新增 _get_overlay_opts() → OverlayOptions（讀四個勾選框狀態）
  - 新增 _refresh_annotated()：從快取 raw/cuts 重繪，不重新載入圖片
  - load_result()：快取 raw/mask/cuts，呼叫 _on_mode_ann()
  - _load_batch_entry()：快取 raw/None/cuts，呼叫 _on_mode_ann()
  - clear()：重設所有快取
  - _on_result_selected()：簡化為呼叫 _refresh_annotated()（包含 focus）
  - 新增模組層級 _ov_chk() helper

【問題3回答 — Y-CD 量測定義（不需修改，僅說明）】

  量測公式（cmg_analyzer.py:116-118）：
    upper_edge = upper.cy + upper.height / 2.0   → 等同於 upper.y1（底部 exclusive）
    lower_edge = lower.cy - lower.height / 2.0   → 等同於 lower.y0（頂部）
    cd_px = lower_edge - upper_edge

  意義：
  - 量的是上方 MG 片段底緣到下方 MG 片段頂緣之間的純空白像素數
  - 即 lower.y0 − upper.y1 = gap 的像素列數
  - 轉換：cd_nm = cd_px × nm_per_pixel

  評估：
  - 這是標準 edge-to-edge gap 量測，定義合理
  - 精度受限於 connected components bounding box（無次像素精度）
  - 若需更高精度可在 Stage 4 加入邊緣梯度精修，現有場景已足夠
